Each project follows a consistent structure: **context → data → analysis → findings → so what**.

## Tech stack

- **Python** — pandas, numpy, scipy
- **Visualisation** — plotly, matplotlib, seaborn
- **SQL** — PostgreSQL, DuckDB for in-notebook querying
- **Notebooks** — Jupyter, with reproducible environments via `requirements.txt`

## Project structure

```
data-analysis-portfolio/
├── 01-project-name/
│   ├── data/           # raw & processed data (gitignored if large)
│   ├── notebooks/      # Jupyter notebooks
│   ├── src/            # helper scripts
│   └── README.md       # project-specific context
├── 02-project-name/
│   └── ...
└── README.md           # this file
```

## Running locally
